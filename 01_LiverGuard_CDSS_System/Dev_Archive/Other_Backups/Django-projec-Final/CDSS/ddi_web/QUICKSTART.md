```bash
cd C:\CDSS\ddi_web

```

## 웹 브라우저에서 접속

```
http://localhost:8000/
```

## 새로운 자동완성 기능 사용 방법

### 1. 실시간 자동완성
약물 이름을 타이핑하기 시작하면 **자동으로** 제안 목록이 나타납니다!

**예시:**
- `A` 입력 → A로 시작하는 모든 약물 표시
- `AS` 입력 → AS로 시작하는 약물 표시 (Aspirin 등)
- `met` 입력 → met으로 시작하는 약물 표시 (Metformin 등)

### 2. 키보드 단축키

#### ⬆️ ⬇️ 화살표 키
- 제안 목록에서 위/아래로 이동

#### Enter 키
- 선택된 약물을 추가
- 선택된 항목이 없으면 첫 번째 항목 자동 선택

#### 마우스
- 제안 항목 위로 마우스를 올리면 자동 선택
- 클릭하여 추가

### 3. 작동 원리

1. **타이핑 시작**
   ```
   입력창에 "a" 입력
   ```

2. **자동 검색** (300ms 후)
   ```
   → A로 시작하는 약물 검색
   → 결과 자동 표시
   ```

3. **선택 및 추가**
   ```
   → 화살표 키로 탐색 또는 마우스 호버
   → Enter 또는 클릭으로 추가
   ```

4. **입력창 자동 초기화**
   ```
   → 다음 약물 검색 준비
   ```

## 사용 예제

### 예제 1: Aspirin 검색

```
1. 입력창 클릭
2. "a" 입력
3. 자동으로 A로 시작하는 약물 목록 표시
4. "s" 추가 입력 → "as"
5. 목록이 좁혀짐 (Aspirin 등)
6. 화살표 키로 "Acetylsalicylic acid" 선택
7. Enter 키 또는 클릭
8. 추가 완료!
```

### 예제 2: Metformin 검색

```
1. "met" 입력
2. "Metformin" 자동 제안
3. Enter 키로 빠르게 추가
```

### 예제 3: 여러 약물 추가

```
1. "aspirin" → Enter
2. "metformin" → Enter
3. "statin" → Enter
4. "위험도 분석" 버튼 클릭
```

## 주요 개선사항

### ✅ 이전 버전
- ❌ 검색 버튼을 눌러야 함
- ❌ 오타 발생 가능
- ❌ 정확한 약물 이름을 알아야 함

### ✨ 현재 버전
- ✅ 타이핑하면 자동으로 제안
- ✅ 오타 최소화
- ✅ 부분 검색 지원 (a, as, asp 모두 가능)
- ✅ 키보드 단축키 지원
- ✅ 첫 글자만 알아도 찾을 수 있음

## 성능 최적화

### 디바운싱 (Debouncing)
- 타이핑할 때마다 즉시 검색하지 않음
- 300ms 대기 후 검색
- 서버 부하 감소 및 속도 향상

### 예시:
```
사용자 타이핑: "a" → "s" → "p" (빠르게)
실제 API 호출: "asp" (한 번만)
```

## API 엔드포인트

자동완성은 다음 API를 사용합니다:

```bash
GET /api/drugs/search/?q=<검색어>
```

**응답 예시:**
```json
{
  "success": true,
  "data": [
    {
      "drug_id": "DB00945",
      "drug_name": "Acetylsalicylic acid"
    },
    {
      "drug_id": "DB00758",
      "drug_name": "Aspirin"
    }
  ]
}
```

## 문제 해결

### 자동완성이 작동하지 않음
1. 브라우저 콘솔 확인 (F12)
2. 네트워크 탭에서 API 호출 확인
3. 서버가 실행 중인지 확인

### 결과가 너무 많이 나옴
- 더 많은 문자를 입력하여 결과 좁히기
- 예: "a" → "as" → "asp"

### 키보드 단축키가 작동하지 않음
- 입력창에 포커스가 있는지 확인
- 브라우저를 새로고침 (F5)
